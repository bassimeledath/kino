## Design Context

### Users
Developers, educators, and content creators who record screencasts, product demos, and tutorials on macOS. They care about output quality — their recordings should look professional without manual editing. They're technically capable, keyboard-savvy, and impatient with friction. They want to hit record, do their thing, and get a polished result fast.

### Brand Personality
**Polished. Fast. Effortless.**

The interface should feel like a precision tool that makes you look good. Every interaction should feel intentional and refined — not flashy, but crafted. The user should feel like the app is doing the hard work (zoom, camera, effects) while they focus on their content.

### Aesthetic Direction
**Native macOS — polished & delightful, fast & minimal.**

Kino should feel like it belongs on macOS. Vibrancy, SF Pro typography, system-grade controls, frosted glass materials, squircle corners. But also fast and minimal — no unnecessary chrome, every pixel earns its place. Think the precision of Screen Studio's HUD with the approachability of Loom's simplicity.

**References:**
- **Screen Studio** — premium macOS HUD, frosted glass toolbar, spring-physics animations, squircle corners, SF Symbol icons, smooth state transitions. The gold standard for macOS recording apps.
- **Loom** — simple, approachable, recording made easy. The "just works" feeling. Low friction from launch to finished recording.

**Anti-references:**
- Generic web apps with Tailwind defaults (rounded-md, zinc buttons, no material)
- Electron apps that feel like websites in a window (no vibrancy, no native feel)
- Over-designed dashboards with too many panels, gradients, and decorative elements
- Windows-style flat UI or Android Material Design

### Design Principles

1. **Native over web.** Use macOS vibrancy, system fonts (SF Pro), and platform conventions. The app should feel like Apple made it, not like a React app in an Electron shell. Prefer `vibrancy: 'under-window'`, backdrop blur, and system-grade controls over CSS approximations.

2. **Quiet confidence.** The UI should be calm and restrained. No attention-seeking animations, no gratuitous color. Let the content (the recording) be the hero. Controls recede until needed. Use subtle shadows, thin borders at low opacity, and muted text hierarchies.

3. **Every interaction feels crafted.** Hover states, press feedback, state transitions — all should use spring physics or eased curves, never instant swaps. Buttons should `active:scale-95` with `transition-all duration-150`. State changes should crossfade or morph, not jump-cut.

4. **Dense but breathable.** Small text (11-13px), compact controls, but with generous internal padding and optical alignment. The settings panel is 280px wide with tight controls — but each control has room to breathe. Think Linear's density with Arc's spacing discipline.

5. **One cursor, one click, done.** Minimize steps. The toolbar should go from launch to recording in one click. The editor should show the result immediately. Export should be one button. Reduce cognitive load — smart defaults over configuration screens.

### Color System
- **Neutrals:** Zinc scale (950 → 50), body at `#0a0a0a`
- **Primary accent:** Red-500 (`#ef4444`) — record, active states, primary CTAs
- **Functional accents:** Blue-500 (playhead), Amber-600 (clips), Teal-500 (manual zoom), Violet-500 (auto zoom)
- **Surfaces:** Semi-transparent zinc with backdrop blur (`bg-zinc-900/80 backdrop-blur-xl`)
- **Borders:** White at 4-8% opacity (`border-white/[0.06]`)
- **Text hierarchy:** White → zinc-300 → zinc-400 → zinc-500 → zinc-600

### Typography
- **Font:** SF Pro (via -apple-system), -0.01em letter spacing, antialiased
- **Scale:** 9px (timestamps) → 11px (labels/controls) → 13px (headers) → 14px (prominent)
- **Weights:** 400 (body), 500 (medium labels), 600 (section headers)
- **Monospace:** SF Mono / system mono for timecodes, tabular-nums for alignment

### Motion
- **Transitions:** 150ms for interactions, 200ms for state changes
- **Press feedback:** `active:scale-95` on all clickable elements
- **Springs:** Hooke's Law springs for camera, cursor, and zoom (F = k*x - d*v)
- **No jarring swaps:** State changes should crossfade or morph, never instant DOM swaps
